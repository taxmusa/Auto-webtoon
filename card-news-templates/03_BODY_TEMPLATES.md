# 03. 본문 슬라이드 템플릿 4종

> 커버 다음에 오는 내용 슬라이드. 세무 정보를 명확하게 전달하는 역할.
> 레퍼런스 이미지에는 본문 슬라이드가 없으므로, 커버의 디자인 톤을 기반으로 설계.
> **각 본문 템플릿은 다크/라이트/블루 3가지 테마 변형을 지원해야 한다.**

---

## 테마 변형 규칙

본문 슬라이드는 커버와 같은 테마를 자동으로 따라간다:

| 커버 템플릿 | 본문 테마 |
|------------|----------|
| COVER-A (다크 BEST) | dark |
| COVER-B (다크 카테고리) | dark |
| COVER-C (블루 리포트) | blue |
| COVER-D (미니멀 카드) | light |
| COVER-E (웜톤 질문) | light (warm 톤) |
| COVER-F (틸 리포트) | dark (teal 톤) |

### 테마별 기본값

BODY_THEMES = {

## BODY-A: 넘버링 리스트형

- **용도**: 포인트별 정보 나열 ("절세 방법 5가지", "체크리스트")
- **비율**: 커버와 동일

### 레이아웃 (3개 항목 기준)

```
┌──────────────────────────────┐
│                              │  ← 테마 bg
│  [슬라이드 번호]    [페이지]   │  ← "01" 크게 / "1/5" 작게
│  ─────────────────────────── │  ← divider
│                              │
│  ❶  [항목 제목 1]             │  ← 번호: accent 컬러, 원형
│      [설명 텍스트 1~2줄]      │     제목: text_primary, body_bold
│                              │     설명: text_secondary, body
│  ❷  [항목 제목 2]             │
│      [설명 텍스트 1~2줄]      │
│                              │
│  ❸  [항목 제목 3]             │
│      [설명 텍스트 1~2줄]      │
│                              │
│  @handle                     │
└──────────────────────────────┘
```

### 넘버링 스타일 옵션

```python
NUMBER_STYLES = {
    "circle_filled": {
        # 채워진 원 안에 숫자
        "bg": "accent",
        "text": "bg",  # 배경색과 같은 색 → 반전 효과
        "size": 48,
        "radius": "50%",
    },
    "text_large": {
        # 큰 숫자만 (01. 02. 03.)
        "text": "accent",
        "size": 56,
        "format": "{:02d}.",  # 01. 02. 03.
    },
    "emoji": {
        # 이모지 (1️⃣ 2️⃣ 3️⃣)
        "format": "emoji",
    },
}
```

### 세무사 적용 예시

```
슬라이드 번호: "01"
항목들:
  1. 제목: "인적공제 꼼꼼히 챙기기"
     설명: "부양가족 기본공제(150만원)와\n추가공제를 빠짐없이 확인하세요."
  2. 제목: "신용카드 공제 최적화"
     설명: "총급여의 25% 초과 사용분부터\n공제가 시작됩니다."
  3. 제목: "연금저축 한도 채우기"
     설명: "연 900만원 한도 IRP 포함,\n최대 148.5만원 세액공제."
```

## BODY-B: 비교형 (Before/After, O/X)

- **용도**: 변경 전후, 올바른/잘못된 비교, 세전/세후
- **비율**: 커버와 동일

### 레이아웃 옵션 1: 좌우 비교

```
┌──────────────────────────────┐
│                              │
│  [비교 주제 제목]              │  ← text_primary, subtitle
│  ─────────────────────────── │
│                              │
│  ┌────────┐   ┌────────┐    │
│  │  BEFORE │   │  AFTER │    │  ← 좌: bad_color / 우: good_color
│  │ ────── │   │ ────── │    │
│  │        │   │        │    │
│  │ 내용1  │   │ 내용1  │    │  ← 비교 항목들
│  │ 내용2  │   │ 내용2  │    │
│  │ 내용3  │   │ 내용3  │    │
│  │        │   │        │    │
│  └────────┘   └────────┘    │  ← card_bg, card_border
│                              │
│  @handle                     │
└──────────────────────────────┘
```

### 레이아웃 옵션 2: 상하 O/X 리스트

```
┌──────────────────────────────┐
│                              │
│  [비교 주제 제목]              │
│  ─────────────────────────── │
│                              │
│  ✕  [잘못된 방법/이전 내용]    │  ← bad_color 아이콘 + 취소선 효과
│     [설명]                   │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  │  ← 점선 구분
│  ✓  [올바른 방법/변경 내용]    │  ← good_color 아이콘
│     [설명]                   │
│                              │
│  💡 [핵심 포인트 한 줄]        │  ← accent 컬러, body_bold
│                              │
│  @handle                     │
└──────────────────────────────┘
```

### 세무사 적용 예시 (좌우 비교)

```
주제: "2025년 증여세 변경사항"
BEFORE (변경 전):
  - "증여재산공제 5,000만원"
  - "세율 10~50% (5단계)"
  - "신고기한 3개월"
AFTER (변경 후):
  - "증여재산공제 5,000만원 (동일)"
  - "세율 10~50% (5단계, 동일)"
  - "신고기한 3개월 (동일)"
```

### 세무사 적용 예시 (O/X)

```
주제: "사업용 경비 처리 O/X"
X: "개인 식비를 사업 경비로 처리"
   "적격증빙 없이 비용 인정 불가"
O: "거래처 접대비로 적격증빙 갖춰 처리"
   "건당 1만원 이하는 간이영수증 가능"
포인트: "경비처리의 핵심은 '업무 관련성' + '적격증빙'입니다"
```

## BODY-C: 하이라이트 박스형

- **용도**: 핵심 숫자/금액 강조, 세금 계산 결과, 절세 효과
- **비율**: 커버와 동일

### 레이아웃

```
┌──────────────────────────────┐
│                              │
│  [소제목]                     │  ← text_secondary, category
│  [메인 설명 제목]              │  ← text_primary, subtitle
│  ─────────────────────────── │
│                              │
│  ┌────────────────────────┐  │
│  │                        │  │  ← highlight 박스 (card_bg)
│  │     [큰 금액/숫자]      │  │  ← number_color, number_xl 폰트
│  │     ₩ 3,450,000       │  │
│  │                        │  │
│  │  [라벨: 예상 절세 금액]  │  │  ← text_secondary, caption
│  │                        │  │
│  └────────────────────────┘  │
│                              │
│  ┌────────────────────────┐  │
│  │  [계산 과정 또는 조건]   │  │  ← 선택적 서브 박스
│  │  항목1: 금액1           │  │     body 폰트, 좌측정렬
│  │  항목2: 금액2           │  │
│  │  ──────────            │  │
│  │  합계:   금액합          │  │  ← body_bold
│  └────────────────────────┘  │
│                              │
│  @handle                     │
└──────────────────────────────┘
```

### 큰 숫자 스타일

```python
HIGHLIGHT_NUMBER = {
    "prefix": "₩",              # 통화 기호 (선택)
    "font": "number_xl",         # 64px Black
    "color": "number_color",     # 테마별 다름
    "format": "{:,.0f}",         # 천단위 콤마
    "suffix": "원",              # 뒤에 붙는 단위 (선택)
    "suffix_size": 36,           # 단위 폰트 크기
}
```

### 세무사 적용 예시

```
소제목: "CASE STUDY"
메인설명: "연봉 5천만원 직장인 절세 효과"
큰숫자: "₩ 1,485,000"
라벨: "연간 예상 절세 금액"
계산과정:
  - "연금저축 (400만원 × 16.5%): 660,000원"
  - "IRP (300만원 × 16.5%): 495,000원"
  - "월세 세액공제 (연 600만원 × 17%): 330,000원"  (선택)
  - "합계: 1,485,000원"
```

## BODY-D: 스텝/프로세스형

- **용도**: 순서가 있는 절차 설명 ("신고 절차 4단계", "신청 방법")
- **비율**: 커버와 동일

### 레이아웃

```
┌──────────────────────────────┐
│                              │
│  [소제목]                     │  ← text_secondary, category
│  [프로세스 제목]               │  ← text_primary, subtitle
│  ─────────────────────────── │
│                              │
│  STEP 1 ──────────────────   │  ← accent 컬러 라벨 + 구분선
│  [단계 제목]                  │  ← text_primary, body_bold
│  [설명 1줄]                   │  ← text_secondary, body
│           │                  │  ← 세로 연결선 (accent, 점선)
│  STEP 2 ──────────────────   │
│  [단계 제목]                  │
│  [설명 1줄]                   │
│           │                  │
│  STEP 3 ──────────────────   │
│  [단계 제목]                  │
│  [설명 1줄]                   │
│           │                  │
│  STEP 4 ──────────────────   │
│  [단계 제목]                  │
│  [설명 1줄]                   │
│                              │
│  @handle                     │
└──────────────────────────────┘
```

### 스텝 연결선 스타일

```python
STEP_CONNECTOR = {
    "type": "dashed_line",   # 점선 세로 연결
    "color": "accent",
    "dash_length": 8,
    "gap_length": 6,
    "thickness": 2,
    "x_offset": 24,          # STEP 라벨 좌측 기준 들여쓰기
}
```

### 세무사 적용 예시

```
소제목: "PROCESS"
프로세스제목: "종합소득세 신고 절차"

STEP 1: "홈택스 접속 및 로그인"
  설명: "공동인증서 또는 간편인증으로 로그인"

STEP 2: "소득 및 공제 내역 확인"
  설명: "근로/사업/기타소득과 공제항목 확인"

STEP 3: "세액 계산 및 신고서 작성"
  설명: "자동계산된 세액 확인 후 신고서 제출"

STEP 4: "납부 또는 환급"
  설명: "추가납부세액은 납부, 환급세액은 계좌로 입금"
```

## 본문 슬라이드 공통 규칙

1. **페이지 표시**: 우측 상단에 "2/7" 형태로 현재 페이지/전체 표시
2. **핸들**: 모든 본문 슬라이드 하단에 @handle 표시
3. **여백**: 커버와 동일한 padding 값 사용
4. **항목 수 제한**: 
   - 1:1 비율: 항목 최대 3개
   - 4:5 비율: 항목 최대 4~5개
   - 항목이 더 많으면 슬라이드 분할
5. **테마 자동 매칭**: 커버 선택 시 본문 테마 자동 결정
