## 1. 캐러셀/카드뉴스 디자인 개선 (핵심)

### 현재 문제점
1. **카드뉴스 배경이 너무 어둡고 칙칙함** (`#1a1a2e` 어두운 남색)
2. **본문 글씨가 너무 작음** (22px → 인스타그램 모바일에서 가독성 떨어짐)
3. **디자인이 단조로움** — 전문 디자이너 수준의 비주얼이 부족
4. **시각적 계층이 약함** — 제목, 본문, 강조 요소의 구분 미흡

### Phase A: 즉시 적용 (기존 함수 파라미터 수정)

#### 1-1. 카드뉴스 기본 색상 개편
**파일**: `app/api/content_generator.py`, `app/api/cardnews.py`

| 항목 | 현재 | 개선 |
|------|------|------|
| 배경색 | `#1a1a2e` (어두운 남색) | `#F8F9FA` (밝은 회색) |
| 텍스트색 | `#FFFFFF` | `#1A1A1A` (진한 회색) |
| 강조색 | `#e94560` (핑크) | `#4A7FB5` (전문적 블루) |

#### 1-2. 글씨 크기 증대
**파일**: `app/services/render_service.py`

| 요소 | 현재 | 개선 |
|------|------|------|
| 제목 (일반) | 44px | **52px** |
| 제목 (커버) | 60px | **68px** |
| 본문 | 22px | **28px** |
| 페이지 번호 | 16px | **20px** |
| 본문 opacity | 0.85 | **1.0** (불투명) |

#### 1-3. Pretendard 폰트 추가
**파일**: `app/services/render_service.py` (KOREAN_FONT_CSS)
- jsDelivr CDN으로 Pretendard 웹폰트 추가
- 기본 font-family: Pretendard → Noto Sans KR → sans-serif

#### 1-4. 그림자 및 라운드 강화
**파일**: `app/services/render_service.py`
- 커버: text-shadow 추가
- 카드: border-radius 24px, box-shadow

### Phase B: 테마 팔레트 시스템

#### 1-5. 5가지 테마 팔레트
**새 파일**: `app/services/theme_palettes.py`

| 테마 | 배경 | 텍스트 | 강조색 | 용도 |
|------|------|--------|--------|------|
| dark | #1A1A1A | #FFFFFF | #FF6B6B | 강렬, SNS 바이럴 |
| light | #FFFFFF | #1A1A1A | #4A7FB5 | 깔끔, 모던 |
| blue | #E8EEF4 | #1A1A1A | #4A7FB5 | 공신력, 리포트 |
| warm | #F5F0EB | #1A1A1A | #8B6914 | 친근, 질문형 |
| teal | #1A3A3A | #FFFFFF | #A8D5D5 | 세련, 트렌디 |

### Phase C: 카드뉴스 전용 템플릿 시스템 (card-news-templates 기반)

#### 1-6. 커버 슬라이드 (6종)
- **COVER-A**: 다크 BEST 리스트형 (강렬, 바이럴)
- **COVER-B**: 다크 카테고리형 (전문적, 긴급성)
- **COVER-C**: 블루 리포트형 (공신력)
- **COVER-D**: 미니멀 카드형 (깔끔, 모던)
- **COVER-E**: 웜톤 질문형 (친근)
- **COVER-F**: 틸 리포트형 (세련)

#### 1-7. 본문 슬라이드 (4종)
- **BODY-A**: 넘버링 리스트형
- **BODY-B**: 비교형 (Before/After, O/X)
- **BODY-C**: 하이라이트 박스형 (핵심 숫자 강조)
- **BODY-D**: 스텝/프로세스형

#### 1-8. 마무리 슬라이드 (2종)
- **CLOSING-A**: 요약 + CTA형
- **CLOSING-B**: 브랜드 카드형

#### 1-9. 템플릿 세트 (사전 조합)
| 세트 | 커버 | 본문 | 마무리 | 테마 |
|------|------|------|--------|------|
| dark_best | COVER-A | BODY-A | CLOSING-A | dark |
| blue_report | COVER-C | BODY-A | CLOSING-A | blue |
| minimal_card | COVER-D | BODY-A | CLOSING-B | light |

## 2. 요약 슬라이드 디자인 개선

### 현재 문제점
- 콘텐츠 많으면 55%까지 자동 축소 → 가독성 치명적
- 글꼴 크기 고정 (26px) → 모바일에서 여전히 작음

### 개선 계획

#### 2-1. 자동 스케일 최소값 상향
**파일**: `app/services/render_service.py`
```
현재: scale 최소 0.55 (55%)
개선: scale 최소 0.75 (75%)
```

#### 2-2. 글꼴 크기 증대

| 요소 | 현재 | 개선 |
|------|------|------|
| 제목 | 36px | **42px** |
| 본문 | 26px | **30px** |
| 테이블 thead | 26px | **30px** |
| 테이블 tbody | 24px | **28px** |
| 체크리스트 | 25px | **28px** |
| Q&A 질문 | 26px | **30px** |
| Q&A 답변 | 24px | **28px** |

## 6. 검증 방법

1. **캐러셀**: 생성 → 밝은 배경, 큰 글씨, 가독성 확인
2. **카드뉴스**: 생성 → 어두운 배경 제거, 프리미엄 디자인 확인
3. **요약 슬라이드**: 6가지 스타일 렌더링 → 글씨 크기, 축소율 확인
4. **테마 전환**: dark/light/blue 각각 생성 → 색상 정상 적용
5. **인스타그램**: 실제 모바일 화면에서 가독성 테스트
