## 1. 이미지 생성 속도 + 안정성 (긴급 - 이미 적용 완료)

### 문제
- 이미지 생성에 138초 소요 (이전 40초, toonstoons 20-30초)
- SDK가 PIL Image를 **PNG로 재인코딩** → 전송량 5배 증가 (2.4MB → 11.9MB)
- 레퍼런스 매 생성마다 리사이징 → 불필요한 오버헤드
- 재시도 로직 과다 (3회 + 최대 26초 대기)

### 적용된 수정

#### 1차 수정 (이전 세션)
| 항목 | 이전 | 변경 후 |
|------|------|---------|
| `responseModalities` | `["IMAGE"]` | **`["TEXT", "IMAGE"]`** (toonstoons 참조) |
| contents 순서 | 레퍼런스 → 프롬프트 | **프롬프트 → 레퍼런스** (toonstoons 참조) |
| 레퍼런스 라벨 | 긴 한국어 설명 | **이미지만 전달** (라벨 제거) |
| 재시도 구조 | 이중 루프 (attempt + retry_count) | **단일 루프** |

#### 2차 수정 (속도 최적화 — 이번 세션)
| 항목 | 이전 | 변경 후 | 효과 |
|------|------|---------|------|
| 이미지 전달 방식 | PIL Image (SDK가 PNG 변환) | **Part.from_bytes (JPEG 직접)** | **전송량 80% 감소** |
| 레퍼런스 리사이징 | 매 생성마다 순차 최적화 | **저장 시점 1회만** | 로드 3-5초 절약 |
| 재시도 횟수 | MAX_RETRIES=3 | **MAX_RETRIES=2** | 최악 23초 절약 |
| 타임아웃 재시도 | 3회 재시도 | **즉시 종료** | 불필요한 대기 제거 |
| IMAGE_GENERATION_TIMEOUT | 90초 | **60초** | 실패 시 빠른 응답 |
| 기존 프리셋 마이그레이션 | 없음 | **migrate_existing_presets()** | 기존 파일 호환 |

### 기대 효과
- **138초 → 30-40초대** (toonstoons 수준)
- 전송량: 11.9MB → 2.4MB (80% 감소)
- 첫 번째 이미지 생성 성공률 대폭 향상

## 2. 캐러셀/카드뉴스 디자인 개선 (핵심)

### 현재 문제점
1. **카드뉴스 배경이 너무 어둡고 칙칙함** (`#1a1a2e` 어두운 남색)
2. **본문 글씨가 너무 작음** (22px → 인스타그램 모바일에서 가독성 떨어짐)
3. **디자인이 단조로움** — 전문 디자이너 수준의 비주얼이 부족
4. **시각적 계층이 약함** — 제목, 본문, 강조 요소의 구분 미흡

### 개선 계획

#### 2-1. 카드뉴스 기본 색상 개편
**파일**: `app/api/content_generator.py` (67-81줄)

| 항목 | 현재 | 개선 |
|------|------|------|
| 배경색 | `#1a1a2e` (어두운 남색) | `#F8F9FA` (밝은 회색) 또는 `#FFFFFF` |
| 텍스트색 | `#FFFFFF` | `#2D3436` (진한 회색) |
| 강조색 | `#e94560` (핑크) | `#FF6B6B` (산호색, toonstoons 참조) |

#### 2-2. 글씨 크기 증대
**파일**: `app/services/render_service.py` (352-538줄)

| 요소 | 현재 | 개선 |
|------|------|------|
| 제목 (일반) | 44px | **52px** |
| 제목 (커버) | 60px | **68px** |
| 본문 | 22px | **28px** |
| 페이지 번호 | 16px | **20px** |
| 본문 opacity | 0.85 | **1.0** (불투명) |

#### 2-3. 프리미엄 디자인 템플릿 추가
**파일**: `app/services/render_service.py`

새 템플릿 추가:
- **`modern-gradient`**: 그라디언트 배경 + 글래스모피즘 카드
- **`minimal-bold`**: 큰 타이포그래피 + 넓은 여백 + 강렬한 컬러 포인트
- **`magazine`**: 잡지 스타일 레이아웃 + 세리프 폰트 + 구분선

#### 2-4. 색상 프리셋 업그레이드
**파일**: `app/api/content_generator.py`

현재 프리셋 시스템에 프로급 프리셋 추가:
```
프리셋명        배경          텍스트       강조색       설명
─────────────────────────────────────────────────────────

ocean-blue     #F0F7FF       #1A365D     #3182CE     시원한 파랑
mint-green     #F0FFF4       #1A4731     #38A169     깔끔한 민트

sunset-gold    #FFFFF0       #4A3728     #D69E2E     황금 노을
```

#### 2-5. 그림자 및 라운드 강화
**파일**: `app/services/render_service.py`

## 3. 요약 슬라이드 디자인 개선

### 현재 문제점
- 콘텐츠 많으면 55%까지 자동 축소 → 가독성 치명적
- 글꼴 크기 고정 (26px) → 모바일에서 여전히 작음
- 제목 스타일이 단조로움

### 개선 계획

#### 3-1. 자동 스케일 최소값 상향
**파일**: `app/services/render_service.py` (871-885줄)

```
현재: scale 최소 0.55 (55%)
개선: scale 최소 0.75 (75%) — 넘치면 페이지 분할
```

#### 3-2. 글꼴 크기 증대
**파일**: `app/services/render_service.py` (610-886줄)

| 요소 | 현재 | 개선 |

| 제목 | 36px | **42px** |
| 본문 | 26px | **30px** |

## 9. 검증 방법

1. **이미지 생성**: 웹툰 이미지 생성 → 첫 번째 씬 성공 여부 확인
2. **캐러셀**: 캐러셀 생성 → 밝은 배경, 큰 글씨, 가독성 확인
3. **카드뉴스**: 카드뉴스 생성 → 어두운 배경 제거, 프리미엄 디자인 확인
4. **요약 슬라이드**: 6가지 스타일 모두 렌더링 → 글씨 크기, 축소율 확인
5. **인스타그램**: 실제 모바일 화면에서 가독성 테스트
