## 2. 핵심 기술 패턴 분석

### 2-1. Text Auto-Resize (텍스트 자동 크기 조절)

**모든 경쟁 서비스가 공통으로 사용하는 핵심 기술.**

#### 동작 원리
```
텍스트 입력 → 컨테이너 크기 확인 → 폰트 크기 계산 → 맞을 때까지 반복
```

#### 구현 방식 3가지

**방식 A: Binary Search (이진 탐색)**
```javascript
// 6px~70px 범위에서 최적 크기를 찾음 (최대 6회 반복)
function findOptimalFontSize(text, container, min=6, max=70) {
    while (max - min > 1) {
        const mid = (min + max) / 2;
        element.style.fontSize = mid + 'px';
        if (element.scrollHeight > container.height) max = mid;
        else min = mid;
    }
    return min;
}
```

**방식 B: CSS clamp() + Container Query (최신 CSS)**
```css
/* 컨테이너 크기에 비례하는 유동 타이포그래피 */
.slide-title {
    font-size: clamp(28px, 5cqi, 72px);  /* 최소 28px, 컨테이너의 5%, 최대 72px */
}
.slide-body {
    font-size: clamp(20px, 3cqi, 36px);
}
```

**방식 C: fitty 라이브러리 (가로 맞춤)**
```javascript
import fitty from 'fitty';
fitty('.slide-title', { maxSize: 72, minSize: 24 });
```

#### 우리 프로그램 현황 vs 경쟁사

| | 우리 | PostNitro | aiCarousels | Tyle |
|--|------|-----------|-------------|------|
| 텍스트 자동 크기 조절 | **없음** (고정 28px) | 있음 | 있음 | 있음 |
| 텍스트 양 감지 | 없음 | AI 분석 | 모드 분류 | 패턴 분석 |
| 컨테이너 맞춤 | 없음 | 자동 | Auto-Resize | 자동 |

**→ 가장 큰 격차. 반드시 구현 필요.**

### 2-2. AI 콘텐츠 구조화

경쟁사들의 AI는 단순히 텍스트를 생성하는 것이 아니라, **슬라이드 구조를 설계**함.

| 역할 | 우리 | 경쟁사 |
|------|------|--------|
| 슬라이드 수 결정 | 사용자 수동 입력 | AI가 내용 깊이 분석 후 자동 결정 |
| 정보 밀도 조절 | 없음 | 슬라이드별 적절한 정보량 배분 |
| 텍스트 길이 제한 | 하드코딩 (20자/100자) | AI가 디자인 공간에 맞게 자동 조절 |
| 핵심 키워드 강조 | 없음 | 자동 볼드/색상 처리 |

### 2-3. 편집 UX 패턴

#### 경쟁사 공통 패턴: "실시간 WYSIWYG"
```
┌───────────────────────────────────────┐
│  [슬라이드 목록]  │  [실시간 미리보기]  │
│   1. 커버         │                     │
│   2. 본문1  ←───→ │   수정하면 즉시     │
│   3. 본문2        │   미리보기에 반영    │
│   4. 마무리       │                     │
├───────────────────┤                     │
│  [편집 도구]      │                     │
│  제목: [입력...]   │                     │
│  본문: [입력...]   │                     │
│  AI: [개선] [축소] │                     │
└───────────────────────────────────────┘
```

**우리 현재 UX 문제:**
1. 편집 → 렌더링 → 미리보기가 3단계 분리
2. 수정할 때마다 Playwright 렌더링 대기 (3~5초/슬라이드)
3. 전체 렌더링이 끝나야 결과 확인 가능

## 3. 우리 프로그램의 구체적 보완점

### 3-1. [최우선] 텍스트 자동 크기 조절 시스템

**현재 문제**: 모든 본문이 28px 고정 → 텍스트 적으면 허전, 많으면 넘침

**개선안: 3단계 적응형 타이포그래피**

```
텍스트 양 측정 → 카테고리 분류 → 해당 프리셋 적용
```

#### 카테고리 분류 기준

| 카테고리 | 본문 글자 수 | 제목 폰트 | 본문 폰트 | 줄간격 | 요소 간격 |
|----------|------------|----------|----------|--------|----------|
| **compact** (많음) | 80자 이상 | 28px | 24px | 1.5 | 16px |
| **normal** (보통) | 40~79자 | 36px | 30px | 1.6 | 24px |
| **spacious** (적음) | 39자 이하 | 44px | 36px | 1.7 | 32px |

#### 구현 위치: template_builders.py

```python
def calculate_text_tier(body_text: str) -> str:
    """텍스트 양에 따른 타이포그래피 등급 계산"""
    char_count = len(body_text.replace('\n', '').replace(' ', ''))
    if char_count >= 80:
        return "compact"
    elif char_count >= 40:
        return "normal"
    else:
        return "spacious"

# 등급별 타이포그래피 프리셋
TEXT_TIERS = {
    "compact":  {"title": 28, "body": 24, "line_height": 1.5, "gap": 16},
    "normal":   {"title": 36, "body": 30, "line_height": 1.6, "gap": 24},
    "spacious": {"title": 44, "body": 36, "line_height": 1.7, "gap": 32},
}
```

이 방식의 장점:
- Playwright 서버 사이드 렌더링과 호환 (JS 라이브러리 불필요)
- 이진 탐색보다 예측 가능하고 빠름
- 디자이너가 프리셋을 세밀하게 튜닝 가능

### 3-2. [최우선] 컨테이너 수직 활용도 개선

**현재 문제**: 본문이 위에서부터 쌓여 하단 50%가 빈 공간

**개선안**:

```css
/* 모든 본문 슬라이드에 적용 */
.slide-content-area {
    display: flex;
    flex-direction: column;
    justify-content: center;      /* 수직 중앙 정렬 */
    min-height: calc(100% - 160px); /* 상하단 여백 제외 */
}

/* 항목이 많을 때는 space-between으로 전환 */
.slide-content-area.many-items {
    justify-content: space-between;
}
```

### 3-3. [높음] 한국어 텍스트 최적화

**현재 문제**: 단어 중간에서 줄바꿈 → 가독성 저하

```css
/* 한국어 최적화 CSS */
.slide-text {
    word-break: keep-all;        /* 한국어 단어 단위 줄바꿈 */
    overflow-wrap: break-word;    /* 긴 단어 강제 줄바꿈 */
    text-wrap: pretty;           /* 미관을 고려한 줄바꿈 (CSS 4) */
}
```

### 3-4. [높음] AI 텍스트 생성 가이드 강화

**현재 문제**: AI가 디자인 제약을 모르고 텍스트를 생성 → 너무 길거나 짧음

**개선안: 프롬프트에 디자인 제약 명시**

```python
# content_generator.py AI 프롬프트 개선
SLIDE_TEXT_GUIDE = """
## 슬라이드 텍스트 작성 규칙:
- 커버: 제목 15자 이내 (2줄 이내), 부제 25자 이내
- 본문: 소제목 12자 이내, 설명 25~50자 (핵심만)
- 마무리: CTA 문구 15자 이내

## 줄바꿈 규칙:
- 의미 단위로 \n 삽입
- 한 줄에 12~18자가 적정
- 숫자/금액은 줄 앞에 배치
- 조사(은/는/이/가)가 줄 끝에 오지 않게

## 예시:
❌ "개인 고객 위주라면 간이과세가 유리! 하지만 초기 인테리어 비용이 많아 부가세 10%를 환급받아야 한다면 일반과세자가 정답입니다."
✅ "개인 고객 위주라면\n간이과세가 유리!\n\n하지만 초기 인테리어 비용이 많아\n부가세 10% 환급이 필요하다면\n일반과세자가 정답입니다."
"""
```

### 3-5. [높음] 편집+미리보기 통합 (실시간 HTML 미리보기)

**경쟁사 공통**: 수정 즉시 미리보기 반영
**우리 현재**: 편집 → 렌더링 버튼 → 3~5초 대기 → 탭 전환해서 확인

**개선안: HTML 실시간 미리보기 + 최종 PNG 렌더링 분리**

```
[편집 중] 텍스트 수정 → 500ms 디바운스 → API에서 HTML만 반환 → iframe으로 표시
[내보내기] "PNG 렌더링" 버튼 → Playwright로 최종 PNG 생성
```

핵심: HTML 생성은 빠름 (1ms), Playwright PNG 렌더링만 느림 (3~5초)
→ 편집 중에는 HTML만 보여주고, 최종 내보내기 시에만 PNG 생성

### 3-6. [중간] 미리보기 → 썸네일 그리드 + 라이트박스

**현재 문제**: 스와이프 뷰에서 1장이 화면 전체 차지 → 전체 흐름 파악 불가

**개선안**: 기본 뷰를 4열 썸네일 그리드로 변경, 클릭 시 라이트박스

### 3-7. [중간] 슬라이드별 AI Writing Assistant

**경쟁사**: aiCarousels, PostNitro 모두 슬라이드별 AI 텍스트 개선 기능 제공

**개선안**: 각 슬라이드 편집 영역에 "AI 개선" 버튼 추가
- 짧게 줄이기 / 다시 쓰기 / 전문적으로 / 친근하게

```
[슬라이드 2 편집]
제목: [간이 vs 일반, 유불리 따져보기]
본문: [개인 고객 위주라면...]
[🤖 AI 개선 ▼] → 짧게 | 다시 쓰기 | 전문적으로 | 친근하게
```

### 3-8. [낮음] 슬라이드 콘텐츠 모드 다양화

**경쟁사** (aiCarousels): 텍스트만 / 텍스트+이미지 / 이미지만 / 스크린샷

**현재**: 텍스트만 (BODY 템플릿 전부 텍스트 기반)

**향후**: 이미지 삽입이 가능한 본문 템플릿 추가 검토

## 5. 권장 구현 순서

### Phase A: 즉시 효과 (난이도 낮음, 1~2시간)
1. 한국어 줄바꿈 CSS (`word-break: keep-all`) — template_builders.py
2. 수직 중앙 정렬 CSS (`justify-content: center`) — template_builders.py
3. AI 텍스트 생성 프롬프트 개선 — content_generator.py

### Phase B: 핵심 차별화 (난이도 중간, 3~4시간)
4. 텍스트 자동 크기 조절 (3단계 적응형) — template_builders.py + theme_palettes.py
5. 프리뷰 이미지 재생성 — generate_previews.py

### Phase C: UX 개편 (난이도 높음, 6~8시간)
6. 편집+미리보기 통합 (HTML 실시간 미리보기)
7. 썸네일 그리드 + 라이트박스
8. AI Writing Assistant
