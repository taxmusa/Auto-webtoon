# 콘텐츠 UI/UX 대개선 계획서

> 작성일: 2026-02-26
> 대상: 카드뉴스/캐러셀 미리보기 + 편집 패널 전면 개편

---

## 현재 문제점 요약

### 문제 1: 미리보기가 화면 전체를 차지
- 스와이프 뷰가 기본값 → 슬라이드 1장이 화면 100% 너비를 차지
- 4:5 카드뉴스의 경우 세로가 매우 길어 스크롤 없이는 버튼도 안 보임
- 전체 슬라이드를 한눈에 볼 수 없어 흐름 파악이 어려움

### 문제 2: 디자인 완성도 부족
- 본문 텍스트가 이미지 크기 대비 작게 느껴짐 (28px / 1080px = 약 2.6%)
- 글 줄바꿈이 의미 단위가 아닌 물리적 너비 기준으로 끊김
- 제목/본문 간 여백, 요소 간 비율이 전문 카드뉴스 대비 부족
- 본문 영역 활용률이 낮음 (하단 빈 공간이 큼)

### 문제 3: 편집과 미리보기가 분리
- ② 편집 → 텍스트 수정 → "이미지 렌더링" 버튼 → ③ 미리보기 전환
- 수정 결과를 보려면 매번 렌더링 + 탭 전환 필요 (왕복 15~30초)
- 실시간 피드백이 없어 시행착오가 많음

**변경**: 썸네일 그리드 기본 → 클릭 시 라이트박스(크게 보기)

│ 160px  │ 160px  │ 160px  │ 160px  │
├────────┼────────┼────────┼────────┤
│ [5/8]  │ [6/8]  │ [7/8]  │ [8/8]  │
│ 본문4  │ 본문5  │ 본문6  │ 마무리 │

**구현 상세**:
- 그리드: `grid-template-columns: repeat(auto-fill, minmax(140px, 1fr))`
- 각 썸네일: `aspect-ratio: 4/5` (카드뉴스) 또는 `1/1` (캐러셀)
- 썸네일 하단에 슬라이드 번호 + 템플릿 타입 표시
- 호버 시: 살짝 확대(scale 1.03) + 그림자 + "🔍 크게 보기" 오버레이

### 2-3. 본문 영역 활용률 개선

**문제**: 본문 내용이 상단에 몰리고 하단 절반이 비어있음

**해결**: 본문 컨테이너에 `justify-content: center` 또는 콘텐츠 양에 따라 자동 간격 분배

```css
/* 현재 */
.slide-body {
    display: flex;
    flex-direction: column;
    /* 위에서부터 쌓임 → 하단 빈 공간 */
}

/* 변경: 항목 수에 따른 적응형 배치 */
.slide-body {
    display: flex;
    flex-direction: column;
    justify-content: space-between;  /* 항목이 적으면 균등 분배 */
    /* 또는 */
    justify-content: center;         /* 수직 중앙 정렬 */
    min-height: calc(100% - header - footer);
}
```

**BODY-A (넘버링 리스트형) 구체적 개선**:
- 항목이 3개 이하일 때: 각 항목의 gap을 40px로 확대 (현재 20px)
- 항목이 4~5개일 때: gap 유지 (20px)
- 번호 배지: 48px → 52px (약간 키움)
- 항목 설명: line-height 1.6 → 1.7

**BODY-B (비교형) 구체적 개선**:
- 2열 카드 간격: 24px → 32px
- 카드 내부 padding: 40px → 48px
- 항목 간 간격: 8px → 14px
- BAD/GOOD 헤더: 20px → 24px

**BODY-C (하이라이트 박스형) 구체적 개선**:
- 큰 숫자: 64px → 80px + letter-spacing: -2px (타이트하게)
- 계산 세부사항: 줄간격 1.8 → 2.0
- 전체 수직 중앙 정렬

**BODY-D (스텝형) 구체적 개선**:
- 스텝 번호: 16px → 20px
- 연결선 높이: 24px → 32px (스텝 간 시각적 연결 강화)
- 스텝 제목/설명 padding-left: 12px → 16px

### 2-4. 텍스트 줄바꿈 개선

**문제**: AI가 생성한 텍스트가 의미 단위로 끊기지 않음
**해결**: AI 텍스트 생성 프롬프트에 줄바꿈 가이드 추가 + CSS word-break 최적화

```css
/* 한국어 텍스트 줄바꿈 최적화 */
.text-content {
    word-break: keep-all;          /* 한국어 단어 단위 줄바꿈 */
    overflow-wrap: break-word;      /* 긴 단어는 강제 줄바꿈 */
    text-wrap: pretty;             /* CSS Text Level 4: 고른 줄바꿈 */
}
```

**AI 프롬프트 개선** (content_generator.py 텍스트 생성 시):
```
- 각 항목의 본문은 20~35자 이내로 작성
- 핵심 키워드나 숫자는 앞쪽에 배치
- 줄바꿈이 필요한 경우 \n으로 의미 단위 구분
- 한 줄에 너무 많은 정보를 넣지 않기
```

### 3-3. 핵심 개념: HTML 실시간 미리보기

**현재 방식**: 텍스트 편집 → API 호출 → Playwright 렌더링 → PNG 이미지 반환 → 표시
**문제**: 매번 서버 왕복 + Playwright 실행 → 슬라이드당 3~5초 대기

**개선 방식**: 텍스트 편집 → **프론트에서 직접 HTML 미리보기** → 즉시 반영
- template_builders.py의 HTML 생성 로직을 **JS로 포팅** (간략 버전)
- 또는 API로 HTML만 받아서 `<iframe>` 또는 `<div>` 안에 표시
- 최종 PNG 렌더링은 "PNG 내보내기" 버튼으로 일괄 처리

**두 가지 구현 방안 비교:**

| | 방안 A: JS 클라이언트 렌더링 | 방안 B: API HTML 반환 + iframe |
|--|--|--|
| **속도** | 즉시 (0ms) | 100~300ms (API 왕복) |
| **정확도** | 근사치 (90~95%) | 정확 (서버와 동일 HTML) |
| **구현 난이도** | 높음 (12종 빌더 JS 포팅) | 중간 (API 엔드포인트 1개 추가) |
| **유지보수** | 서버/클라이언트 동기화 필요 | 서버만 수정하면 됨 |
| **권장** | | **✅ 방안 B 권장** |

### 3-4. 방안 B 상세: API HTML 미리보기

#### 새 API 엔드포인트

```
POST /api/content/preview-html
Request:
{
    "template_set": "dark_best",
    "template_type": "BODY-A",    // 또는 자동 배정

    "total_slides": 8,

    "height": 1350
}

Response:
{
    "html": "<div style='width:1080px; height:1350px; ...'>...</div>"
}
```

#### 프론트엔드 미리보기 표시

```html
<!-- 미리보기 영역 -->
<div id="live-preview-container" style="width:100%; max-width:400px;">
    <iframe id="live-preview-frame"
        style="width:1080px; height:1350px;

               transform-origin: top left;
               border: none; pointer-events: none;"
        sandbox="allow-same-origin">
    </iframe>
</div>
```

#### 디바운스 실시간 업데이트

```javascript
let previewTimeout = null;

function onEditChange() {
    clearTimeout(previewTimeout);
    previewTimeout = setTimeout(async () => {
        const html = await fetchPreviewHTML(currentSlideIndex);
        const frame = document.getElementById('live-preview-frame');
        frame.srcdoc = html;
    }, 500);  // 500ms 디바운스 — 타이핑 중에는 요청 안 함
}
```

- 템플릿 타입 드롭다운: 변경 시 즉시 미리보기 갱신

### 3-7. 단계 구조 변경

```
현재: ① 설정  │  ② 편집  │  ③ 미리보기   (3단계)
변경: ① 설정  │  ② 편집+미리보기           (2단계)
```

- "이미지 렌더링" 버튼 → "🖼️ PNG 렌더링 (전체)" 로 이름 변경
  - 이 버튼을 누를 때만 Playwright로 실제 PNG 생성
  - 렌더링 완료 후 자동으로 썸네일 그리드(개선안 1) 표시
- 그리드에서 개별 슬라이드 클릭 → 라이트박스로 최종 PNG 확인

### 3-8. 반응형 레이아웃 (모바일 대응)

```
[데스크탑 (≥1024px)]
좌: 슬라이드 목록 (250px) | 우: 미리보기 (나머지)

```

## 구현 순서 (Phase)

### Phase A: 디자인 품질 향상 (독립 — 먼저 적용 가능)

1. `theme_palettes.py` — TYPOGRAPHY / LAYOUT 상수 조정
2. `template_builders.py` — CSS 개선 (word-break, justify-content, gap)
3. `content_generator.py` — AI 텍스트 생성 프롬프트 가이드 추가
4. 프리뷰 이미지 재생성 (`generate_previews.py` 실행)

**예상 영향**: 기존 워크플로우 변경 없이 렌더링 품질만 향상

### Phase B: 미리보기 UI 개편 (썸네일 + 라이트박스)

1. `index.html` — 스와이프 뷰 제거, 그리드를 기본 뷰로
2. `index.html` — 라이트박스 컴포넌트 추가 (모달 오버레이)
3. `index.html` JS — 스와이프 함수 → 라이트박스 함수 교체
4. 그리드 썸네일 크기/호버 효과 조정

**예상 영향**: 미리보기 사용성 대폭 개선, 편집 로직 영향 없음

### Phase C: 편집+미리보기 통합 (핵심 개편)

1. `content_generator.py` — `POST /api/content/preview-html` API 추가
2. `index.html` — 3단계 → 2단계 구조 변경
3. `index.html` — 통합 패널 레이아웃 (슬라이드 목록 + 미리보기 + 편집)
4. `index.html` JS — 디바운스 실시간 미리보기 + 슬라이드 선택 로직
5. 반응형 CSS 추가

**예상 영향**: UX 전면 개편, 가장 큰 변경

## 최종 사용 흐름 (개선 후)

```
1. 사용자가 ① 설정에서 주제, 템플릿 세트, 슬라이드 수 입력
2. "AI 텍스트 생성" 클릭 → AI가 슬라이드별 제목/본문 생성
3. 자동으로 ② 편집+미리보기 탭으로 전환
4. 좌측 슬라이드 목록에서 원하는 슬라이드 클릭
5. 우측에 해당 슬라이드의 HTML 미리보기가 표시됨
6. 하단 편집 영역에서 텍스트 수정 → 0.5초 후 미리보기 자동 갱신
7. 템플릿 타입 변경 → 즉시 미리보기 갱신
8. 모든 슬라이드 확인 후 "🖼️ PNG 렌더링" 클릭
9. Playwright가 전체 슬라이드를 PNG로 렌더링
10. 썸네일 그리드로 전체 결과 확인
11. 클릭하면 라이트박스로 크게 보기
12. "발행탭으로 이동" → Instagram 발행
```

## 리스크 및 고려사항

| 리스크 | 대응 |
|--------|------|
| iframe 미리보기와 실제 PNG 렌더링 차이 | 동일 HTML 사용, 폰트 로딩 확인 |

| HTML 미리보기 API 추가 부하 | build_template_slide()만 호출, Playwright 미사용 → 부하 미미 |
